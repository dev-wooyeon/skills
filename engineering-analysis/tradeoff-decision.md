# tradeoff-decision

## 역할

기술 또는 구조 선택지를 장단점 수준에서 끝내지 않고, 현재 제약과 리스크를
기준으로 어떤 선택이 더 맞는지 판단 근거를 만드는 스킬입니다.

## 언제 쓰는가

- 두 개 이상의 기술이나 구조를 비교할 때
- 설계 이유를 문서로 남겨야 할 때
- 정답보다 현재 맥락에 맞는 선택을 설명해야 할 때

## 입력

- 비교 대상
- 현재 시스템 상황
- 제약 조건

## 출력

- 비교 기준
- 선택지별 장점과 한계
- 현재 상황에서의 적합성
- 최종 선택과 비선택 이유
- 감수해야 할 리스크

## 핵심 규칙

- 비교 기준을 먼저 정의합니다.
- 현재 상황의 제약을 명시합니다.
- 선택 이유와 선택하지 않은 이유를 함께 씁니다.
- 만능 정답이 아니라 맥락 적합성을 설명합니다.

## 관리 포인트

- 비교 기준이 실제 팀의 의사결정 방식과 어긋나지 않는지 봅니다.
- 유행하는 기술 비교가 아니라 문제와 제약 중심으로 유지되는지 확인합니다.
