# problem-structuring

## 역할

복잡하게 얽힌 문제를 증상, 근본 원인, 반복 요인, 영향 범위, 해결 가능한
영역으로 나누어 설명 가능한 형태로 바꾸는 스킬입니다.

## 언제 쓰는가

- 문제가 크지만 어디부터 손대야 할지 모를 때
- 기술 문제와 운영/프로세스 문제가 섞여 있을 때
- 장애나 운영 이슈를 문서화 가능한 형태로 정리해야 할 때

## 입력

- 문제 상황 설명
- 관련 로그, 사례, 운영 이슈

## 출력

- 문제 정의
- 증상
- 근본 원인
- 반복 요인
- 영향 범위
- 해결 옵션과 우선순위

## 핵심 규칙

- 문제를 한 문장으로 먼저 정의합니다.
- 해결책보다 증상과 원인을 먼저 분리합니다.
- 기술, 프로세스, 운영으로 해결 영역을 나눕니다.

## 관리 포인트

- 감정 표현보다 검증 가능한 사실 중심 원칙이 유지되는지 확인합니다.
- 산출물 형식이 실제 회고나 운영 정리 문서와 잘 맞는지 점검합니다.
