# technical-writing-type-selector

## 역할

기술 문서가 학습 중심, 문제 해결, 참조, 설명 중 어떤 유형으로 작성되어야
하는지 판단하는 스킬입니다.

## 언제 쓰는가

- 문서 방향이 아직 정해지지 않았을 때
- 독자 수준과 문서 목표에 맞는 형식을 먼저 정해야 할 때
- 하나의 문서에 너무 많은 목적이 섞여 있는지 점검할 때

## 입력

- 문서 목표
- 독자 수준
- 프로젝트 상황
- 추가 고려 사항

## 출력

- 추천 문서 유형
- 추천 이유
- 보조 유형
- 작성 시 유의할 점
- 문서 구성 제안

## 핵심 판단 기준

- 초급 독자와 체계적 학습이면 `학습 중심`으로 기웁니다.
- 즉시 실행과 빠른 해결이 중요하면 `문제 해결`로 기웁니다.
- 반복 조회와 정확한 lookup이 중요하면 `참조`를 고릅니다.
- 배경, 원리, 비교, 설득이 중요하면 `설명`을 고릅니다.

## 관리 포인트

- 문서 유형 정의가 팀의 실제 글쓰기 기준과 어긋나지 않는지 점검합니다.
- 하이브리드 허용 규칙이 과도해지지 않도록 한 문서 한 목표 원칙을
  유지합니다.

## 실행 프롬프트

```text
$technical-writing-type-selector

문서 목표: <예: React Hook의 개념을 이해시키고 싶다>
독자 수준: <예: React를 처음 접하는 초급 개발자>
프로젝트 상황: <예: 새 기술 온보딩 문서를 만들고 있다>
추가 고려 사항: <예: 짧은 시간 안에 초안을 완성해야 한다>

위 정보를 바탕으로
1. 가장 적합한 주 문서 유형을 하나 고르고
2. 왜 그 유형이 맞는지 설명하고
3. 필요하면 보조 유형을 하나만 제안하고
4. 그 유형에 맞는 작성 시 유의점과 권장 문서 구성을 제안해줘.
```
