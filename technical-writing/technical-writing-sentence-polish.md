# technical-writing-sentence-polish

## 역할

기술 문장을 더 짧고 명확하며 일관되게 다듬는 문장 단위 스킬입니다.
문서 구조 변경보다 표현 개선에 집중합니다.

## 언제 쓰는가

- 문장이 길고 복잡해서 읽기 속도가 떨어질 때
- 메타 표현이나 모호한 표현이 많을 때
- 용어가 섞이거나 주체가 불분명할 때

## 입력

- 한 문장
- 짧은 문단
- 여러 개의 어색한 기술 문장

## 출력

- 문장 진단
- 핵심 문제
- 수정 방향
- 다듬은 문장

## 핵심 편집 원칙

- 필요한 정보만 남깁니다.
- 메타 담화를 줄입니다.
- 구체적이고 직접적으로 씁니다.
- 용어를 일관되게 씁니다.
- 독자나 개발자를 주체로 드러냅니다.

## 관리 포인트

- 문장 압축이 의미 손실로 이어지지 않는지 확인합니다.
- 능동형 전환 기준과 용어 정규화 기준이 일관되게 유지되는지 봅니다.

## 실행 프롬프트

```text
$technical-writing-sentence-polish

입력 문장:
<다듬고 싶은 문장 또는 짧은 문단>

용어 제약:
<반드시 유지할 용어가 있으면 적기>

톤 제약:
<예: 기술 문서 톤, 짧고 단정하게>

위 문장을 기준으로
1. 문장 진단
2. 핵심 문제
3. 수정 방향
4. 여러 개선 요소를 반영한 하나의 최종 문장
형식으로 정리해줘.
```
